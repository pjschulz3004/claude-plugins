Step 0 — Check for forge.local.md:
```bash
[ -f forge.local.md ] && echo "exists" || echo "missing"
```

**If $ARGUMENTS is `new` OR forge.local.md is missing:**
- Create `forge.local.md` at project root with full initial YAML frontmatter (phase: IGNITE, step: SPEAK, all gate flags false, agent_xp: [], project: set to current directory name via `basename $(pwd)`). Include `# Forge Pipeline State` header and a "Do not edit manually" note in the body.
- Inform user: "Starting Forge IGNITE phase."
- Execute the forge-ignite workflow from `@$HOME/.claude/get-shit-done/workflows/forge-ignite.md`

**If $ARGUMENTS is `status`:**
- Read forge.local.md
- Display a formatted dashboard showing:
  - Project name and current phase + step
  - Gate flags for the current phase (true = passed, false = pending) as a checklist
  - Agent XP log (last 5 entries if any)
  - Time hint: "Resume with: /gsd:forge"

**If $ARGUMENTS is `reset`:**
- Use AskUserQuestion to confirm: "This will clear all Forge pipeline state for this project. Are you sure? (yes/no)"
- If yes: `rm forge.local.md` and display "Forge state cleared. Run /gsd:forge new to start fresh."
- If no: display "Reset cancelled."

**If $ARGUMENTS is empty AND forge.local.md exists:**
- Read `phase` field from forge.local.md
- Route based on phase value:
  - `IGNITE` → execute forge-ignite workflow
  - `TEMPER` → execute forge-temper workflow
  - `DELIVER` → execute forge-deliver workflow
  - `COMPLETE` → display summary from forge.local.md, suggest `/gsd:forge status`, note pipeline is complete

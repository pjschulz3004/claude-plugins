  <behavior>
    - PromptVersionStore creates `prompt_versions` table: id, task_name, version (int), prompt_text, status (current|candidate|retired), created_at
    - PromptVersionStore creates `prompt_version_metrics` table: id, prompt_version_id, run_id (FK to task_runs.id), success (boolean), duration_ms, tokens (input+output)
    - recordMetric(taskName, version, runId, success, durationMs, tokens) inserts a row
    - getMetrics(taskName, version) returns { runs: number, successRate: number, avgDuration: number, avgTokens: number }
    - PromptVersioner.parseVersion(yamlContent, taskName) extracts version number from `# version: N` comment above or inline with the prompt block
    - PromptVersioner.selectPrompt(taskName) returns the prompt to use: alternates between current and candidate if a candidate exists, based on run count parity
    - PromptVersioner.registerCandidate(taskName, promptText, version) stores a new candidate version
    - PromptVersioner.evaluate(taskName, minRuns=10) compares current vs candidate metrics: if candidate has higher success rate (or equal success rate + fewer tokens), promote candidate; otherwise revert candidate to retired
    - PromptVersioner.promote(taskName) swaps candidate to current, current to retired, updates heartbeat.yaml on disk
    - Add `# version: 1` comment to each task prompt in heartbeat.yaml
  </behavior>
  <action>
    **PromptVersionStore** (packages/jarvis-daemon/src/state/prompt-versions.ts):
    Create a class that takes a `Database.Database` instance (shared from TaskLedger.database, per D-02). Two tables:
    - `prompt_versions`: id INTEGER PK, task_name TEXT, version INTEGER, prompt_text TEXT, status TEXT CHECK(IN 'current','candidate','retired'), created_at TEXT
    - `prompt_version_metrics`: id INTEGER PK, prompt_version_id INTEGER FK, run_id INTEGER, success INTEGER (0/1), duration_ms INTEGER, tokens INTEGER
    - Index on (task_name, status) for fast lookups
    - Methods: registerVersion(), recordMetric(), getMetrics(), getCurrentVersion(), getCandidateVersion(), updateStatus()

    **PromptVersioner** (packages/jarvis-daemon/src/prompt-versioner.ts):
    Create a class that takes PromptVersionStore + yamlPath. Methods:
    - `parseVersion(taskName)`: Read heartbeat.yaml, find `# version: N` comment for the task, return N
    - `selectPrompt(taskName)`: If no candidate exists, return current prompt. If candidate exists, count total runs for both; if runs are even, return current; if odd, return candidate. This gives true alternation.
    - `registerCandidate(taskName, promptText)`: Increment version, store as candidate
    - `evaluate(taskName, minRuns)`: Get metrics for current and candidate. If both have >= minRuns runs, compare success rate first (higher wins), then avg tokens (lower wins as tiebreak). Return {winner: 'current'|'candidate', reason: string}
    - `promote(taskName)`: Set candidate to 'current', old current to 'retired'. Read heartbeat.yaml, replace prompt text and version comment for the task, write back.
    - `revert(taskName)`: Set candidate to 'retired', log the reversion.

    **heartbeat.yaml**: Add `# version: 1` as first line of each task's prompt block.

    Use the yaml library (already a dependency) for parsing. When writing back, use yaml.stringify to maintain structure.

    Per D-01: version is a comment in the YAML, not a separate field.
    Per D-02: metrics stored in SQLite alongside task_runs.
  </action>

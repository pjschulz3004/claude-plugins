<objective>
Wire the growth engine's council review and regression detection into the nightly loop, and enrich the reflection prompt so each claude -p round has mission context, the improve skill procedure, telemetry data, and explicit instructions to run tests before committing.

Purpose: The skeleton loop dispatches rounds but has no quality gates. Without council review, bad changes ship unchecked. Without regression detection, correction rates can silently worsen. Without a richer prompt, the claude -p session lacks the procedure and context to do meaningful work.

Output: A fully wired growth.ts that runs council after each round's implementation, reverts on council rejection or regression, and a co-located test file.
</objective>

    After each successful `dispatcher.dispatch` call, add the council + regression gate:

       ```typescript
       const headBefore = gitExec(["rev-parse", "HEAD"], cfg.repoRoot);
       // ... dispatch happens ...
       const headAfter = gitExec(["rev-parse", "HEAD"], cfg.repoRoot);
       const madeCommit = headBefore !== headAfter;
       ```

    3. **If madeCommit, get diff and test results:**
       ```typescript
       const diff = gitExec(["diff", "HEAD~1", "HEAD"], cfg.repoRoot);
       let testResults: string;
       try {
         testResults = execFileSync("npm", ["test"], { cwd: cfg.repoRoot, timeout: 60_000 }).toString();
       ```

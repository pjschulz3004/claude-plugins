    - storeCorrectionEpisode(correction: CorrectionEvent) creates an Episode: subject={name:correction.task_name, type:"task"}, relation={type:"CORRECTED_BY", properties:{original, corrected, type}}, object={name:"correction_{date}", type:"correction"}
    - searchContext(keywords: string[]) calls kg.search() for each keyword, deduplicates, returns formatted string for prompt inclusion (max 5 results)

    **KGBridge** (packages/jarvis-daemon/src/kg-bridge.ts):

    Create a class that wraps KnowledgeGraphClient with growth-specific convenience methods:

    ```typescript
    export class KGBridge {
      constructor(private readonly kg: KnowledgeGraphClient | null) {}
    ```

    1. `async storeGrowthEpisode(roundNumber: number, description: string, commitHash: string, filesChanged: string[]): Promise<void>`
       - Build an Episode per D-02: subject is the growth session entity, object is the improvement entity
       - Include round number, commit hash, date, and files changed in properties
       - source: "growth_engine"
       - Per KG-01

    2. `async storeCorrectionEpisode(correction: CorrectionEvent): Promise<void>`
       - Build an Episode per D-03: subject is the task entity, object is the correction entity
       - Include original_decision, corrected_decision, correction_type in relation properties
       - Timestamp from correction.detected_at
       - source: "correction_detection"
       - Per KG-02

    3. `async searchContext(keywords: string[], maxResults: number = 5): Promise<string>`
       - For each keyword, call `kg.search(keyword)`
       - Deduplicate by entity name
       - Format as a readable summary: "Previous improvements: ..." and "Past corrections: ..."
       - Cap at maxResults total entries
       - Return empty string if no results or KG unavailable
       - Per KG-03

    4. Graceful degradation: if `this.kg` is null, all methods return immediately (no error). Per D-05 (existing jarvis-kg pattern).

    **Tests** use a mock KnowledgeGraphClient (no real Neo4j needed):
    - Mock addEpisode to capture calls and verify Episode structure
    - Mock search to return canned results and verify formatting
    - Test null KG client returns gracefully

**skills/triage/SKILL.md** (per ORCH-05) — Email triage classification procedure. Frontmatter: name "triage", description "Classify unread emails using deterministic rules first, LLM fallback for ambiguous. Deterministic-first reduces cost and improves consistency."

Body — step-by-step procedure:

Step 1: Load Rules — Read `${CLAUDE_PLUGIN_ROOT}/references/email-rules.md`. Parse the sender rules table, invoice keywords, and category definitions into working memory.

Step 2: Fetch Unread — Call `mcp__jarvis-email__list_unread` (limit 25). For each email, extract: uid, sender (full address), subject, date, flags.

Step 3: Deterministic Classification (for each email, in priority order):
  - Signal 1 — Sender match: extract email address from sender field, check against sender rules table. If match found, assign category and auto_delete value.
  - Signal 2 — List-Unsubscribe: if the email has this header (check via search with header filter if available, or infer from sender patterns like "noreply@", "newsletter@", "digest@" combined with bulk sender indicators), classify as newsletter.
  - Signal 3 — Invoice keywords: check subject for: rechnung, invoice, faktur, beleg, receipt, order confirmation, quittung, zahlungsbestatigung. If found and subject also mentions PDF/attached/attachment, classify as invoice.
  - Signal 4 — No match: mark as "ambiguous" for LLM classification.

Step 4: LLM Classification (for ambiguous emails only) — For each ambiguous email, classify using the LLM fallback instructions from email-rules.md. Consider: is it from a real person? Is it a reply? Is it automated reference info? Is it bulk marketing? When uncertain, prefer reference over noise. Output: category + reason.

Step 5: Route — For each classified email:
  - Determine business vs personal: if to/from it@jschulz.org -> business, else personal.
  - Determine target folder per email-rules.md routing table.
  - If category is noise: call `mcp__jarvis-email__trash`.
  - If target folder exists: call `mcp__jarvis-email__move` to target folder.
  - If auto_delete set: call `mcp__jarvis-email__set_keyword` with $AutoDelete3d or $AutoDelete7d.
  - If category is action_required: call `mcp__jarvis-email__flag`.

Step 6: Summary — Produce a triage summary grouped by category (action_required first, noise last). Include: count per category, subject lines for action_required and invoice categories, total emails processed. Follow `${CLAUDE_PLUGIN_ROOT}/references/jarvis-voice.md` tone.

**skills/brief/SKILL.md** (per ORCH-06) — Cross-domain briefing synthesis. Frontmatter: name "brief", description "Synthesize cross-domain data into a natural-language briefing. Combines calendar, email, budget, todos, and files."

Body — step-by-step procedure:

Step 1: Read voice reference from `${CLAUDE_PLUGIN_ROOT}/references/jarvis-voice.md`.

Step 2: Gather data (call all in sequence):
  - Calendar: `mcp__jarvis-calendar__list_events` for today. Note event times, summaries, locations.
  - Todos: `mcp__jarvis-calendar__list_todos` for pending items. Note overdue ones.
  - Email: `mcp__jarvis-email__list_unread` (limit 10). Count total, note action-required subjects.
  - Budget: `mcp__jarvis-budget__get_categories`. Note overspent categories and total available.
  - Files: `mcp__jarvis-files__list_inbox`. Count pending files.

Step 3: Cross-reference — Find connections:
  - Match email senders to today's calendar attendees
  - Flag overdue todos if calendar is light (suggest tackling them)
  - Connect invoices to budget categories
  - Note any files that relate to today's meetings

Step 4: Synthesise — Write the briefing following voice rules:
  - Lead with the most urgent/important item
  - Group by priority, not by domain
  - Be specific: "3 unread emails, 1 from Sarah about the contract review" not "some emails"
  - Include cross-domain connections naturally
  - End with a light observation or suggestion if appropriate (dry wit, not forced)
  - Target 10-20 lines

**skills/filing/SKILL.md** (per ORCH-07) — Invoice extraction and filing. Frontmatter: name "filing", description "Extract PDF attachments from invoice emails, smart-name them, and file to inbox for archival."

Body — step-by-step procedure:

Step 1: Find invoice emails — Call `mcp__jarvis-email__search` with folder "INBOX/Business/Invoices" (or search for flagged invoice emails).

Step 2: For each invoice email:
  - Extract sender name (company), date, and amount from subject/body
  - Construct smart filename: `YYYY-MM-DD_{company}_{amount}.pdf` (e.g. `2026-03-15_Hetzner_4.51EUR.pdf`)
  - If the email has a PDF attachment, save it to inbox via `mcp__jarvis-files__save_to_inbox`

Step 3: Archive processed emails — Mark processed invoice emails as read via `mcp__jarvis-email__mark_read`.

Step 4: Summary — Report: N invoices processed, filenames created, any that couldn't be processed (missing attachment, unparseable amount). Follow voice reference tone.

Note: This skill runs weekly (Monday 9am heartbeat task). PDF extraction depends on the email MCP server exposing attachment content — if not available, the skill should flag emails for manual download and still perform the smart-naming for the filename convention.

**Auth middleware (TG-05):**
- First middleware on the bot: check `ctx.chat?.id.toString() === config.chatId`
- If unauthorized, silently return (do NOT reply -- avoid revealing the bot exists)
- Log unauthorized attempts at warn level: `[telegram] Unauthorized message from chat ${ctx.chat?.id}`

**Error handler (Pitfall 6):**
- `bot.catch((err, ctx) => { console.error('[telegram] Unhandled error:', err); })` -- log, do NOT rethrow
- Every individual command/handler also wrapped in try/catch that sends user a brief error message: "Something went wrong. I'll look into it."

**7 Slash commands (TG-01):**

1. `/start` -- Reply with "Jarvis at your service. Use /status for system health, or just send me a message."
2. `/status` -- Import BreakerManager.getAll() and ledger.getRecent() for last 5 tasks. Format: uptime (process.uptime()), breaker states, last 5 task results with status emoji (check/cross).
3. `/inbox` -- Import `@jarvis/email` backend directly. Call listUnread(10). Format as numbered list: "1. From: sender - Subject (date)". Wrap in try/catch; on failure reply "Email service unavailable."
4. `/today` -- Import `@jarvis/calendar` backend directly. Call listEvents for today. Format: "09:00-10:00 Meeting with X\n...". If no events: "Clear schedule today."
5. `/budget` -- Import `@jarvis/budget` backend directly. Call getSummary(). Format top 5 categories by activity with balances.
6. `/tasks` -- Import `@jarvis/calendar` backend directly (VTODOs). Call listTodos(). Format as checklist.
7. `/history` -- Query ledger.getRecent("*", 10) or all task names. Show last 10 task runs across all tasks with name, status, duration.

For /inbox, /today, /budget, /tasks: these import tool backends directly to avoid claude -p overhead (per CONTEXT.md and Pitfall 3). Use try/catch for each; on CircuitBreaker open, reply "Service temporarily unavailable."

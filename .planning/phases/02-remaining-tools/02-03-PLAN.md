<!-- YNAB SDK — key API surface -->
ynab package provides:
- ynab.API class: constructor(accessToken: string)
- api.budgets.getBudgets() -> { data: { budgets: Budget[] } }
- api.categories.getCategories(budgetId) -> { data: { category_groups: CategoryGroup[] } }
- api.transactions.getTransactions(budgetId, sinceDate?) -> { data: { transactions: TransactionDetail[] } }
- api.transactions.updateTransaction(budgetId, transactionId, { transaction: { category_id } })
- api.transactions.updateTransactions(budgetId, { transactions: [{ id, approved: true }] })
- CRITICAL: All IDs from YNAB SDK are UUID objects, NOT strings. Must call String(id) on every ID before returning to callers.
- Amounts are in milliunits (divide by 1000 for display)

  <action>
    1. Create package.json for @jarvis/budget with dependencies: @jarvis/shared, @modelcontextprotocol/sdk, ynab, zod. Same structure as jarvis-email/package.json.
    2. Create tsconfig.json with composite:true, referencing ../jarvis-shared.
    3. Create src/types.ts with:
       - YNABConfig interface: { accessToken: string, budgetId: string }
       - BudgetCategory interface: { id: string, name: string, groupName: string, budgeted: number, activity: number, balance: number }
       - Transaction interface: { id: string, date: string, amount: number, payee: string, categoryName?: string, categoryId?: string, memo?: string, approved: boolean, cleared: string }
       - Zod schemas:
         * GetCategoriesInputSchema ({}) — no params
         * GetTransactionsInputSchema ({ startDate?: z.string(), endDate?: z.string() })
         * CategorizeTransactionInputSchema ({ transactionId: z.string(), categoryId: z.string() })
         * ApproveTransactionsInputSchema ({ transactionIds: z.array(z.string()) })
    4. Create src/backend.ts with:
       - BudgetBackend interface: getCategories(), getTransactions(startDate?, endDate?), categorizeTransaction(transactionId, categoryId), approveTransactions(transactionIds)
       - YnabBackend class implementing BudgetBackend
       - Constructor takes YNABConfig and optional retryDelayMs
       - Private createApi(): returns new ynab.API(this.config.accessToken) — fresh client per operation (connection-per-operation pattern; YNAB SDK is stateless HTTP so this is lightweight)
       - Private withRetry<T>(operation): wraps in retry loop for transient errors (ECONNRESET, ETIMEDOUT, ECONNREFUSED, EPIPE, EAI_AGAIN), 3 attempts, exponential backoff. Auth errors (401, 403) NOT retried.
       - CRITICAL UUID-to-string conversion (BUD-07): Every ID field from YNAB responses MUST be wrapped in String(). The YNAB SDK returns UUID objects that look like strings in console.log but fail when used as object keys or in strict comparisons. Apply String() when mapping response data to our types.
       - CRITICAL milliunit conversion: All amount fields from YNAB are in milliunits (1000 = $1.00). Divide by 1000 when mapping to our types.
       - getCategories: api.categories.getCategories(budgetId), flatten category_groups into BudgetCategory[], filter out hidden and deleted, convert IDs with String(), convert amounts from milliunits
       - getTransactions: api.transactions.getTransactions(budgetId, startDate), filter by endDate if provided (client-side), map to Transaction[] with String() on all IDs, milliunit conversion on amounts
       - categorizeTransaction: api.transactions.updateTransaction(budgetId, transactionId, { transaction: { category_id: categoryId } })
       - approveTransactions: api.transactions.updateTransactions(budgetId, { transactions: transactionIds.map(id => ({ id, approved: true })) })
    5. Create src/index.ts re-exporting types and backend.
    6. Write tests FIRST (TDD):
       - src/types.test.ts: Zod schema validation tests
       - src/backend.test.ts: Mock ynab module. Test getCategories returns flattened, filtered categories with String IDs and dollar amounts. Test getTransactions maps correctly with UUID-to-string. Test categorizeTransaction calls updateTransaction. Test approveTransactions calls updateTransactions. Test retry on network error, no retry on 401.
    7. Use `vi.mock('ynab')` to mock the YNAB API class.

    Key YNAB pitfall from Python Jarvis: UUID objects from YNAB SDK cause SQLite binding errors and comparison failures if not converted to strings. This was a production bug that took hours to diagnose. Apply String() defensively on every ID.

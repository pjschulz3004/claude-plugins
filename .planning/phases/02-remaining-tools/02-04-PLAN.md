  <behavior>
    - FilesConfig validates with baseDir string
    - FileEntry has name, path (relative to baseDir), size (bytes), modified (ISO date string), location ("inbox"|"outbox"|"archive")
    - LocalFilesBackend.listInbox() returns FileEntry[] for files in {baseDir}/inbox/
    - LocalFilesBackend.listOutbox() returns FileEntry[] for files in {baseDir}/outbox/
    - LocalFilesBackend.saveToInbox(filename, content) writes file to {baseDir}/inbox/{filename}, creates inbox dir if missing
    - LocalFilesBackend.moveToOutbox(filename) moves file from inbox to outbox, creates outbox dir if missing
    - LocalFilesBackend.archiveFile(filename) moves file from outbox to {baseDir}/archive/YYYY/MM/{filename} using current date, creates archive dir if missing
    - LocalFilesBackend.syncWebdav() runs rclone sync via execFile (safe, no shell), returns stdout
    - Path traversal attacks prevented: filenames must not contain .. or /
    - Missing directories are auto-created (mkdir -p equivalent)
  </behavior>
  <action>
    1. Create package.json for @jarvis/files with dependencies: @jarvis/shared, @modelcontextprotocol/sdk, zod. NO external file libraries -- uses only node:fs/promises, node:child_process (execFile), node:path. Same package.json structure as jarvis-email.
    2. Create tsconfig.json with composite:true, referencing ../jarvis-shared.
    3. Create src/types.ts with:
       - FilesConfig interface: { baseDir: string }
       - FileEntry interface: { name: string, path: string, size: number, modified: string, location: "inbox" | "outbox" | "archive" }
       - Zod schemas:
         * ListFilesInputSchema ({}) -- no params
         * SaveToInboxInputSchema ({ filename: z.string(), content: z.string() })
         * MoveToOutboxInputSchema ({ filename: z.string() })
         * ArchiveFileInputSchema ({ filename: z.string() })
         * SyncWebdavInputSchema ({}) -- no params
    4. Create src/backend.ts with:
       - FilesBackend interface: listInbox(), listOutbox(), saveToInbox(filename, content), moveToOutbox(filename), archiveFile(filename), syncWebdav()
       - LocalFilesBackend class implementing FilesBackend
       - Constructor takes FilesConfig
       - Private validateFilename(name): throws if name contains "..", "/", or "\" (path traversal prevention)
       - Private ensureDir(dirPath): calls fs.mkdir(dirPath, { recursive: true })
       - Private listDir(location: "inbox"|"outbox"): fs.readdir with stat for each file, returns FileEntry[] sorted by modified desc
       - listInbox(): listDir("inbox") on {baseDir}/inbox/
       - listOutbox(): listDir("outbox") on {baseDir}/outbox/
       - saveToInbox(filename, content): validateFilename, ensureDir inbox, fs.writeFile to {baseDir}/inbox/{filename}
       - moveToOutbox(filename): validateFilename, ensureDir outbox, fs.rename from inbox/{filename} to outbox/{filename}
       - archiveFile(filename): validateFilename, compute YYYY/MM from current date, ensureDir archive/YYYY/MM, fs.rename from outbox/{filename} to archive/YYYY/MM/{filename}
       - syncWebdav(): use node:child_process.execFile (NOT exec -- prevents shell injection) to spawn `rclone` with args `["sync", path.join(baseDir, "outbox"), "mailbox:"]`. Wrap in promisify. Timeout 60s. Throw on non-zero exit. Return stdout.
       - No retry logic needed for local filesystem operations (instant, no network). Retry only on syncWebdav (3 attempts for network errors).
    5. Create src/index.ts re-exporting types and backend.
    6. Write tests FIRST (TDD):
       - src/types.test.ts: Zod schema validation tests
       - src/backend.test.ts: Use real temp directories (node:os tmpdir + fs.mkdtemp). Test listInbox returns files. Test saveToInbox creates file. Test moveToOutbox moves file. Test archiveFile creates YYYY/MM structure. Test validateFilename rejects "../evil". Test syncWebdav mocks execFile from node:child_process. Clean up temp dirs in afterEach.
    7. Backend tests use real filesystem (tmpdir) for file operations -- no mocking needed except for rclone subprocess.
  </action>
